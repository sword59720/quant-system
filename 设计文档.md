## 3. 系统总流程

### 3.1 统一入口
- 脚本：`scripts/run_quant.py`
- 工作流构建：`core/workflows.py::build_run_all_steps`
- 默认步骤（不跳过抓数/报告时）：
  1. 拉取数据：`stock_etf`、`stock_single`、`crypto`
  2. 运行模型：`run_stock_etf.py`、`run_stock_single.py`、`run_crypto.py`
  3. 报告与通知：`paper_forward_stock.py`、`report_execution_quality.py`、`notify_execution_quality_wecom.py`

### 3.2 核心设计原则
- 先产出“目标仓位”（targets），再由交易脚本生成“订单差量”（trades）
- 风控分层：
  - 算法内风险预算（波动/回撤/趋势）
  - 覆盖层（overlay）与执行拦截（execution guard）
  - 数据可用性与缓存兜底（data guard）

## 4. ETF策略设计（当前主股票策略）

### 4.1 策略入口与模式
- 脚本：`scripts/stock_etf/run_stock_etf.py`
- 当前模式：`mode: global_momentum`
- 主要函数：`run_global_momentum(runtime, stock, risk)`

### 4.2 数据输入
- 数据文件：`data/stock/<symbol>.csv`
- 抓数脚本：`scripts/stock_etf/fetch_stock_etf_data.py`
- 抓数源优先级：
  1. Baostock
  2. AkShare EastMoney ETF接口
  3. AkShare Sina接口
- 缓存兜底：若新抓取失败但本地缓存满足最小行数/最大时效，则继续使用缓存

### 4.3 信号与打分逻辑
- 核心资产：
  - 风险资产池：`universe`
  - 基准：`benchmark_symbol`（当前 `510300`）
  - 防守资产：`defensive_symbol`（当前 `518880`）
- 资产打分：
  - 基础：`score = momentum / volatility`
  - 结构化升级（当前开启）：
    - 多窗口动量加权
    - 相对基准动量
    - 资产趋势过滤（相对资产MA）
- 市场状态（regime）：
  - 依据基准趋势（价格 vs MA）+ 广度（trend & momentum为正的占比）
  - 状态：`risk_on / neutral / risk_off`
  - Phase2（当前开启）可基于短长波动比率触发状态降级

### 4.4 仓位生成与预算
- 有效总仓位链路：
  1. `base_alloc`（`capital_alloc_pct`）
  2. `exposure_gate`（CPCV最差超额收益门控，可能下调）
  3. regime乘数（`regime_profiles`）
  4. 结构化乘数（趋势强度、广度）
  5. phase2乘数（波动状态、回撤恢复）
  6. signal_strength乘数（可选，当前关闭）
  7. risk_governor乘数（目标波动、回撤阈值、动量阈值）
- 资产内分配：
  - 先按得分选 TopN（可在phase2自适应调整）
  - 权重由“逆波动权重 + 得分权重混合”得到
  - 应用单标的上限（默认来自 `risk.position_limits.stock_single_max_pct`，支持 symbol cap 覆盖）
  - 剩余仓位补到防守资产

### 4.5 交易前风控层
- `risk_overlay`（当前开启）：
  - 监控 `outputs/reports/stock_paper_forward_latest.json`
  - 若超额收益/策略回撤触发阈值，直接切换防守仓，并强制再平衡
- `execution_guard`（当前开启）：
  - 最小再平衡间隔：`min_rebalance_days`
  - 最小换手阈值：`min_turnover`（可动态）
  - 若不满足则 `hold`，沿用上次目标仓位
- `universe_change_guard`：
  - 支持对股票池变化分级（none/small/medium/large）
  - 可触发回测门禁/CPCV门禁及告警（取决于配置）

### 4.6 输出
- 目标仓位：`outputs/orders/stock_targets.json`
- 状态：`outputs/state/stock_signal_state.json`、`outputs/state/stock_risk_overlay_state.json`
- 股票池变化报告：`outputs/reports/stock_universe_change_latest.json`

## 5. Crypto策略设计（当前为合约长短组合）

### 5.1 策略入口与模型选择
- 脚本：`scripts/crypto/run_crypto.py`
- 模型解析：`core/crypto_model.py::resolve_crypto_model_cfg`
- 因 `trade.market_type: swap`，当前使用 `contract_model` 配置
- 当前引擎：`engine: advanced_ls_rmm`

### 5.2 数据输入
- 数据文件：`data/crypto/<symbol>.csv`
- 抓数脚本：`scripts/crypto/fetch_crypto_data.py`
- 当前交易所模式：`exchange: htx_direct`
- 数据抓取策略：
  - HTX直连为主，支持分页历史拉取
  - 历史模式可回退 OKX REST / ccxt
  - 代理策略可配置（auto/env/direct）
  - DNS受限时可按“就绪缓存数量”软降级继续运行

### 5.3 信号与状态判定
- 每币种特征：
  - 加权动量（当前窗口 120/240，权重 1.0/0.0）
  - 年化波动率
  - 趋势开关（价格 vs MA窗口）
  - 评分：`score = momentum / annualized_vol`
- 多空候选：
  - 多头：得分最高的 `top_n`
  - 空头：得分最低的 `short_top_n`
- 市场状态 `regime_state`：
  - 根据全市场平均动量与趋势广度判定 `risk_on / neutral / risk_off`

### 5.4 仓位构建（advanced_ls_rmm）
- 多空预算：
  - 当前固定预算：`ls_long_alloc_pct=0.40`、`ls_short_alloc_pct=0.30`
  - `ls_dynamic_budget_enabled=false`（动态预算关闭）
- 头寸分配：
  - 候选内按逆波动分配
  - 单标的上限：默认 `risk.position_limits.crypto_single_max_pct`（当前 0.30）
- 置信度缩放（当前开启）：
  - 使用 `score_spread` 将整体仓位缩放到 `[0,1]`
- 风险管理杠杆（当前开启）：
  - 估计组合年化波动
  - 目标波动 0.20，杠杆限制 `[0.2, 1.0]`
- 总暴露上限：
  - `max_exposure_pct`（当前 1.5）限制总绝对仓位

### 5.5 自动下单逻辑（当前已开启）
- 条件：`runtime.env=live` 且 `crypto.execution.auto_place_order=true`
- 流程：
  1. `crypto_targets.json` -> `generate_trades_crypto.py`
  2. 过滤小额订单（`min_order_notional`）
  3. `execute_trades_crypto.py` 实际下单
- 合约动作映射：
  - `OPEN_LONG / CLOSE_LONG / OPEN_SHORT / CLOSE_SHORT`

### 5.6 输出
- 目标仓位：`outputs/orders/crypto_targets.json`
- 交易指令：`outputs/orders/crypto_trades.json`
- 执行记录：`outputs/orders/execution_record_*.json`

## 6. Stock Single策略设计（当前关闭）

### 6.1 任务编排
- 脚本：`scripts/stock_single/run_stock_single.py`
- 任务：
  - `pool`：建池
  - `risk`：5分钟快风控
  - `hourly`：小时信号
  - `full`：`pool -> risk -> hourly`

### 6.2 建池逻辑（build_pool）
- 输入：`pool.source_file` 或 fallback
- 过滤：
  - A股代码合法性
  - 硬过滤（ST、历史长度、停牌特征、近端涨跌停异常）
  - 流动性过滤（20日成交额/成交量）
- 打分：复用 `scoring_core.compute_score_bundle`
  - 因子含动量、反转、波动、估值、主力资金流、情绪、布林特征
  - 横截面z-score后按权重聚合
- 入池出池：
  - `entry_quantile` / `exit_quantile`
  - `min_pool_hold_days`
  - 行业分散约束（`max_industry_weight` + `min_industry_count`）
- 输出：`outputs/orders/stock_single_pool.json`

### 6.3 小时信号（eval_hourly）
- 基于池内最新分数快照生成 BUY/SELL/HOLD
- 阈值模式：`threshold / quantile / topk`（当前 `topk`）
- 买入信号附带：
  - `entry_price`（buy buffer）
  - `stop_price`（ATR止损）
  - `exit_price`（ATR止盈）
- 风控联动：
  - 若快风控触发 `block_new_buys`，BUY降级为HOLD
  - 若 `force_sell_symbols` 命中，直接SELL
- 输出：`outputs/orders/stock_single_signals.json`

### 6.4 快风控（risk_check_fast）
- 输入：`data/stock_single/intraday_risk_latest.csv`
- 触发条件：
  - 组合5分钟收益跌破阈值
  - 单标的5分钟急跌
  - 波动zscore异常
- 动作：
  - 阻断新增买入
  - 对崩跌标的强制卖出（可配置）
- 输出：
  - `outputs/orders/stock_single_risk_state.json`
  - `outputs/orders/stock_single_risk_alerts.json`

### 6.5 QMT执行桥
- 脚本：`scripts/stock_single/qmt_stock_single_strategy.py`
- 支持：
  - `plan`：仅生成订单计划
  - `live`：通过 `xtquant` 下单

## 9. 关键设计结论（当前版本）
- 当前生产主线为“双引擎”：
  - 股票：ETF全局动量 + 多层风控与执行拦截
  - 币圈：合约多空（advanced_ls_rmm）+ 波动目标杠杆 + 自动下单
- `stock_single` 已具备完整链路，但配置上默认关闭，不参与默认实盘主流程。
- 风险控制不是单点，而是“信号内预算 + 覆盖层 + 执行层 + 数据层容错”的串联结构。
